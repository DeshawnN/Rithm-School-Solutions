Functions: Hoisting

  1. What does the following code output? Why? 

    var firstName = 'Elie';

function displayFirstName(){
    var firstName = 'Tim';
    return firstName;
}

displayFirstName(); // ?

    It would display "Tim"

  2. What does the following code output? Why? 

    function displayFirstName(){
    var firstName = 'Tim';
    return firstName;
}

displayFirstName();

console.log(firstName) // ?

    It would throw an ReferenceError stating that firstName isn't defined, because firstName doesn't exist in the global scope, where console.log() is being invoked.

  3. What does the following code output? Why? 

    console.log(firstName) // ?
var firstName = "Elie"

    It would output undefined, because its assignment statement is after the console.log().

  4. What does the following code output? Why? 

    console.log(firstName) // ?
firstName = 'Elie'

    It would output a ReferenceError stating that firstName isn't defined, this is because without the var keyword the firstName variable is merely being assigned, not declared, and since it isn't being declared it doesn't get hoisted.

  5. What does the following code output? Why? 

    function sayHi(){
    return 'Hi ' + firstName;
    var firstName = 'Elie'
}

sayHi(); // ?

  6. What does the following code output? Why? 

    function sayHi(){
    return 'Hi ' + firstName; 
    firstName = 'Elie'
}

sayHi(); // ?

    It will display "Hi undefined", because at the point of returning firstName is declared but with a value of undefined. It's also worth noting that anything after return is skipped anyway

  7. What does the following code output? Why? 

    sayHi() // ?

function sayHi(){
    return 'Hi!';
}

    "Hi" is displayed

  8. What does the following code output? Why? 

    sayHi() // ?

var sayHi = function(){
    return 'Hi!';
}

    It will throw a TypeError, because at the time of invocation, sayHi is a variable with a value of undefined, rather than a function
