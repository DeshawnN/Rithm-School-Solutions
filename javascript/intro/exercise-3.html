    Answer: true

  2. 2 === 2;

    Answer: true

  3. 10 % 3;

    Answer: 1

    Answer: true

  5. true && false;

    Answer: false

Part II

Answer the following questions about this code block:

		var isLearning = true;

		if(isLearning){
		    console.log("Keep it up!");
		} else {
		    console.log("Pretty sure you are learning....");
		}
  1. What should the above code console.log?

    "Keep it up!"

  2. Why do we not need to specify if(isLearning === true)? Why does if(isLearning) work on its own?

    Because by default, the conditional will trigger so long as the value or expression is truthy, which isLearning is. the foo === true expression while helpful when starting to code serves little purpose outside of being visual training wheels.

		var firstVariable;
		var secondVariable = "";
		var thirdVariable = 1;
		var secretMessage = "Shh!";

		if(firstVariable){
		    console.log("first");
		} else if(firstVariable || secondVariable){
		    console.log("second");
		} else if(firstVariable || thirdVariable){
		    console.log("third");
		} else {
		    console.log("fourth");
		}
  1. What should the above code console.log? Why?

    "third"

  2. What is the value of firstVariable when it is initialized?

    undefined

  3. Is the value of firstVariable a "truthy" value? Why?

    No, It's undefined, making it a falsy value.

  4. Is the value of secondVariable a "truthy" value? Why?

    No, as with the above its designated as a falsy value, due to it being an empty string (or rather, a string with a length of 0).

  5. Is the value of thirdVariable a "truthy" value? Why?

    Yes!, unlike the above two values, 1 is infact a truthy value, as 1 can be coerced into true (since it's a number greater than 0).
