  2. See what happens when you have multiple variables of the same name. Which one takes precedence?

    The one defined later in the scope is the one that takes precedence, this is due to the fact that when a variable is defined, it creates a namespace for it to occupy. Namespaces are basically a unique name that points to the data stored in said variable making it impossible for 2 variables to share a name within the same scope, so when a new variable is declared with an existing name, It's treated simply as an assigment of a new value to the pre-existing namespace.

  5. What is the difference between null and undefined?

    null is the the intentional absense of data, which is to say, a variable that that has been declared and initialized to be treated as if it has no value at all, while undefined simply means that the the variable may or may not be declared, and if it is, it currently doesn't have a value stored.

  6. What is NaN in JavaScript? What is the typeof NaN?

    NaN in JavaScript (and some other programming languages) is any non-number value. typeof NaN is "number", because ironically enough, despite NaN meaning not-a-number its still of the numerical data type.
